EmporiUm - East Region Sales Project Summary

For this project, I looked at sales data for the East region to better understand how the territory is performing. The goal was to use SQL to pull the right information from the database, then use Excel to turn the results into charts and a dashboard that would be easier to explain.



I started by connecting the sales table to the store location table and the management table. This was important because the sales table showed the transaction information, but the other tables helped connect each sale to a store, state, and region. Once the data was connected, I could filter everything to only show the East region.



The first thing I checked was the total revenue for the East region and the date range of the data. This gave me a quick overview of how much money the region made and what time period the sales covered. After that, I broke the revenue down by month to see how sales changed over time.



I also looked at product categories to see how many transactions happened each month and what the average transaction size was. This helped show which categories were getting activity and which ones may have higher-value sales. Then I ranked the stores by total revenue so I could see which locations were performing the strongest.



For the Excel part, I used the query results to create a dashboard. I included charts for revenue by state and top stores by revenue. I also made the dashboard interactive so it would be easier to filter and compare results.



The main takeaway from the project is that the sales team should focus on the stores and areas that are already performing well, while also reviewing lower-performing stores to see where they need support. This could include better inventory, promotions, local marketing, or extra sales attention.

Overall, this project helped me practice connecting tables in SQL, using functions like SUM, COUNT, AVG, MIN, and MAX, and turning raw data into a clear business recommendation.
